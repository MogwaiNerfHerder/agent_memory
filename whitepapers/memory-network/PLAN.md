# Memory Network Whitepaper — Plan

## Working title
**The Memory Network: Why AI-Native Organizational Memory Is the First Foundation for AI Value**

## Audience
Mid-market and enterprise executives evaluating "AI strategy" — economic buyers, COOs, CIOs, CROs. Not engineers. They need to come away believing:
1. Their current document/search infrastructure cannot deliver the AI value being promised.
2. The first thing to build is not a chatbot — it is institutional memory.
3. They can take action this quarter, not after a 12-month transformation.

## Themes the user specified
- Why keyword search at scale fails (the "Cortado Group personas" problem)
- The "AI Sherpa" framing: AI as guide through your organization's terrain, not a generic oracle
- Why memory networks are an *early* value — what compounds and what doesn't
- Distribution: MCP / agent API / chat — chosen by audience savvy, not by what's technically cool
- How the work is done — explained for executive confidence, not developer credibility

## Phased approach
1. **Folder + plan + outline** ← we are here
2. **MECE review of the outline** — confirm sections don't overlap and don't leave gaps
3. **Research pass** — pull citations for: knowledge graphs in enterprise, retrieval failure modes, AI adoption gating factors, "human in the loop" / Sherpa metaphors, MCP / agent-API momentum
4. **Section drafts** — written one at a time so the voice stays consistent
5. **Continuity + MECE pass** — re-read end-to-end; check transitions, eliminate duplication, confirm each section earns its place
6. **Polish** — exec summary, opening hook, closing call-to-action, visual placeholders

## Tone
Executive, opinionated, specific. Cortado has *built* this — the paper should sound like a field report, not a vision deck. Concrete examples > abstract framing. No "AI is transforming everything" sentences.

## Deliverable structure
```
whitepapers/memory-network/
├── PLAN.md           ← this file
├── OUTLINE.md        ← section-by-section skeleton with key beats
├── PAPER.md          ← final assembled draft
├── drafts/           ← per-section drafts during writing
├── research/         ← citation notes, source URLs
└── figures/          ← diagram sketches / screenshots
```

## Out of scope
- Implementation details below the architectural level (no schema diagrams, no code)
- Cortado-internal team / org details (this is for external readers)
- Comparison shopping vs specific vendors (we describe the category, not the bake-off)
